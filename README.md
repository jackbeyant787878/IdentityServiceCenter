# IdentityService 分布式权限中心 · README

**IdentityService** 是一款基于 \.NET 10 构建的轻量化、高可扩展、分布式统一权限认证与授权中心。项目严格遵循 DDD 领域驱动设计、CQRS 读写分离架构及 SOLID 编程原则，基于 OpenIddict 6\.0\+ 实现标准 OAuth2\.0/OIDC 协议，支持 SaaS 账号密码认证、微服务间服务授权、双 Token 令牌机制，专为分布式微服务架构体系提供统一、安全、标准化的身份认证与权限管控能力。

✅ **核心定位**：统一身份认证、分布式权限管控、微服务安全网关、SaaS 多租户身份底座

✅ **适用场景**：微服务集群、SaaS 多租户系统、前后端分离项目、分布式企业级系统

## 🏗️ 架构设计

本项目摒弃传统分层架构的臃肿耦合问题，采用**DDD 领域驱动设计 \+ CQRS 读写分离 \+ SOLID 设计原则**三位一体的架构体系，实现高内聚、低耦合、可扩展、可维护的企业级权限服务架构。

### 1\. DDD 领域驱动设计

以「权限、身份、租户、角色、用户」为核心领域模型，拆分边界清晰的业务领域，彻底解决传统权限系统业务混杂、职责模糊的问题。

- **领域层（Domain）**：核心业务实体（用户、角色、权限、租户）、领域规则、领域事件，不依赖任何外部框架，纯业务逻辑实现，保证业务稳定性。

- **应用层（Application）**：协调领域逻辑，处理请求编排，不包含核心业务规则，依赖 MediatR 实现事件解耦。包含所有 **Command / Query**业务用例。

- **基础设施层（Infrastructure）**：数据库持久化、缓存实现、第三方服务对接、令牌生成与校验、OpenIddict 适配。

- **展示层（Presentation / HttpApi）**：\.NET 10 Web API 控制器，仅做协议接收、参数转发、结果响应，**零业务逻辑**。

同时基于 DDD 领域事件机制，实现用户创建、角色变更、权限更新、租户配置变更等场景的异步通知，支撑分布式系统数据同步。

### 2\. CQRS 读写分离架构

严格区分**读操作（Query）**与**写操作（Command）**，彻底解决读写争抢、性能瓶颈、业务复杂度叠加问题，与当前源码结构完全对齐。

- **Command 写操作**：Token 交换、令牌吊销、客户端注册、登出等变更类操作，保证数据一致性、事务完整性。

- **Query 读操作**：用户身份信息查询、权限解析等查询场景，支持缓存加速。

通过 MediatR 中介者模式实现 CQRS 调度，控制器不直接调用 Service、不写业务，全部交由 Application 层处理，实现**请求与业务完全解耦**。

### 3\. SOLID 编程原则落地

项目全程遵循 SOLID 五大设计原则，结合 C\# 14 新特性优化代码结构，适配当前控制器与业务代码结构：

- **单一职责**：ConnectController 只处理 OIDC 协议端点，ClientApplicationController 只处理客户端注册，职责完全隔离。

- **开闭原则**：新增登录模式、新增授权类型只需扩展 Command，无需修改控制器接口。

- **里氏替换**：所有基础设施实现可无缝替换（数据库、缓存、令牌服务）。

- **接口隔离**：细粒度 IRepository、业务服务接口，无冗余依赖。

- **依赖倒置**：控制器依赖 MediatR 抽象，不依赖具体业务实现。

## 🔐 核心授权认证机制

系统基于标准 **OAuth2\.0 / OIDC 1\.0** 协议，依托 OpenIddict 6\.0\+ 构建标准化、可定制的认证授权体系，兼顾通用性与业务定制化能力，同时适配 SaaS 租户场景与微服务集群场景。

### 1\. 核心协议能力

- **OAuth2\.0**：主流的第三方授权、资源访问授权协议，解决微服务之间、客户端与服务端之间的资源授权问题。

- **OIDC（OpenID Connect）**：基于 OAuth2\.0 扩展的身份认证协议，在授权基础上增加用户身份信息标准化传输，实现统一身份登录。

OpenIddict 作为底层引擎，全权接管令牌签发、校验、吊销、过期状态机，业务层只需要处理**用户合法性、租户合法性、权限装载**。

### 2\. 完整认证授权链路（源码对应链路）

1. **协议接收**：ConnectController\.Token 接口接收标准 OIDC 表单请求

2. **业务校验**：ExchangeTokenCommand 完成账号、租户、风控、密码校验

3. **身份装载**：生成 ClaimsPrincipal 身份主体

4. **协议签发**：OpenIddict 输出标准 AccessToken / RefreshToken

5. **下游鉴权**：网关/微服务校验 Token，调用 UserInfo 接口获取可信用户信息

6. **安全销毁**：支持主动 Revoke 吊销、Logout 注销

## 🎫 令牌体系：JWT \+ 双 Token 机制

系统采用 OpenIddict 原生 JWT 生成机制，严格实现 **AccessToken \+ RefreshToken 双令牌模型**，适配金融/支付场景高安全要求。

### 1\. AccessToken（短期业务令牌）

- 短有效期，用于所有业务接口鉴权

- 承载用户、租户、门店、角色、权限声明

- 泄露风险极低，适合前端存储

### 2\. RefreshToken（长期刷新令牌）

- 仅用于刷新令牌，不参与业务鉴权

- 持久化存储，支持 **主动吊销**

- 支撑用户无感续期，无需重复登录

### 3\. 安全熔断能力（源码独有）

支持运行中强制吊销任意 Token，适配风控系统“紧急红色按钮”能力，解决传统 JWT 无法作废的致命短板。

## 💻 核心接口源码落地原理（完全对齐工程代码）

整个 HttpApi 层只有两个控制器，**零业务逻辑、零 SQL、零缓存操作**，严格遵循 CQRS \+ 中介者架构，是标准企业级 DDD 落地范例。

### 1\. ConnectController（标准 OIDC 四大协议端点）

该控制器为系统对外核心安全入口，100% 遵循 OIDC 协议规范，适配浏览器登录、服务鉴权、风控熔断、单点登出。

#### 1\.1 Token 令牌交换接口 /api/connect/token

- **请求约束**：强制 `application/x-www-form-urlencoded`，标准 OIDC 表单提交，防止非法 Json 攻击

- **源码流程**：接收 OpenIddict 标准请求 \-\> 转发 **ExchangeTokenCommand** \-\> 根据业务结果返回登录/禁止/错误

- **成功逻辑**：返回 SignIn 身份凭证，OpenIddict 自动生成双 Token

- **失败逻辑**：区分 Forbid 禁止登录、参数错误、账号风控冻结

- **设计亮点**：协议层与业务层完全隔离，支持任意授权模式扩展

#### 1\.2 UserInfo 用户可信信息接口 /api/connect/userinfo

- **鉴权级别**：必须携带合法有效的 OpenIddict AccessToken

- **核心作用**：下游微服务/支付网关**二次可信校验**，防止伪造本地 Token

- **源码逻辑**：从 Token 解析 subject\(用户ID\) \-\> 执行 **GetUserInfoQuery** \-\> 校验用户激活状态 \-\> 返回标准化用户声明

- **返回结构**：sub、username、user\_type、merchant\_id、store\_id、status，适配网关统一解析

- **风控能力**：自动拦截冻结、注销、风控黑名单用户

#### 1\.3 Revoke 令牌紧急吊销接口 /api/connect/revoke

- **业务定位**：支付风控系统紧急熔断入口

- **能力**：支持吊销 AccessToken / RefreshToken

- **原理**：依托 OpenIddict 状态机，自动清理缓存与数据库令牌记录，秒级失效

- **适用场景**：密钥泄露、终端被盗、商户欠费、操作员异常

#### 1\.4 Logout 安全登出接口 /api/connect/logout

- **双层注销机制**：清空本地 Cookie 登录态 \+ 销毁 OpenIddict 服务端 SSO 凭证

- **效果**：彻底销毁登录上下文，杜绝单点登录残留

### 2\. ClientApplicationManagementController（客户端注册管理）

用于动态接入 SaaS 应用、微服务客户端，实现集群服务准入管控。

#### 2\.1 应用注册接口 /api/clientapplicationmanagement/register

- **入参模型**：AppName、ClientType、TenantId（租户隔离）

- **源码流程**：DTO 接收 \-\> 封装 **RegisterClientApplicationCommand** \-\> MediatR 调度生成客户端凭证

- **安全机制**：客户端密钥仅首次返回，线上需开启平台管理员权限策略

- **场景**：新微服务接入、商户自研应用注册、第三方信任系统接入

### 3\. 工程统一架构规范（源码体现）

- **无业务控制器**：所有逻辑下沉 Command/Query，可单独单元测试

- **协议标准化**：完全对齐 OIDC 规范，兼容所有标准客户端

- **强安全隔离**：用户登录、服务授权、令牌吊销、客户端注册权限隔离

- **金融级风控**：支持主动令牌作废、账号状态拦截、租户资源隔离

## 🚀 未来 CICD 与分布式集群架构规划

为适配生产级分布式高可用、自动化运维、统一流量管控需求，项目后续将落地 **GitHub Actions 自动化流水线 \+ 腾讯云 K3s 容器部署 \+ API 网关统一鉴权 \+ 分布式服务注册中心** 全套云原生 DevOps 架构，实现服务标准化、自动化、可扩容、高可用部署。

### 1\. CICD 自动化部署方案（GitHub Actions \+ 腾讯云 K3s）

全程依托 GitHub Actions 构建全自动 CI/CD 流水线，通过 SSH 远程部署至**腾讯云 K3s 轻量级 Kubernetes 集群**。K3s 可理解为**精简简易版 K8s**，裁剪了 K8s 冗余、过期、非核心组件，保留全套生产可用的核心能力（容器编排、副本扩容、滚动更新、负载均衡、集群自愈、资源调度等），兼容标准 K8s 所有 API 与资源清单。具备极低资源消耗、部署极简、运维成本低的特点，完美适配腾讯云轻量服务器、中小规模生产集群，完全满足本项目微服务集群部署、自动化迭代、多实例高可用运行需求。无需手动干预，适配迭代发布、版本回滚、多实例扩容场景。

- **持续集成 CI**：代码提交/合并触发自动构建、代码校验、单元测试、镜像打包、推送私有镜像仓库，杜绝不规范代码上线。

- **持续部署 CD**：通过 GitHub Actions SSH 远程连接腾讯云服务器，自动更新 K3s 部署配置、滚动更新 Pod 实例，实现**零停机发布**。

- **多实例高可用部署**：K3s 集群配置多 Pod 副本部署策略，支持负载均衡、故障自动漂移、弹性扩容，解决单点故障问题，适配高并发鉴权场景。

- **版本管控与回滚**：流水线保留历史版本，支持一键回滚，保障生产环境发布安全稳定。

### 2\. 标准生产级架构：API 网关 \+ 注册中心 内外流量隔离体系

本项目后续完整落地**业界标准微服务生产架构**，严格区分 **外网前端流量** 与**内网微服务点对点流量**，实现内外网隔离、安全分层、性能分层、治理分层，完全契合主流 Spring Cloud / \.NET Cloud 企业级架构规范。

#### 2\.1 内网微服务通信：去中心化点对点调用（高性能、无网关转发）

微服务与微服务之间的内部调用，采用**去中心化点对点直连模式**，不经过 API 网关转发，最大化内网通信速度，适配高频、高吞吐服务内部交互场景。

- **调用模式**：ServiceA 直接内网调用 ServiceB，纯内网链路，网络损耗极低、响应速度快。

- **注册发现机制**：所有微服务（含 IdentityService 权限中心、业务服务、任务服务）**必须统一注册到分布式注册中心**（Consul/Eureka/Nacos）。
    服务提供者：启动时主动注册服务信息（服务名、IP、端口、健康状态）至注册中心。

- 服务消费者：启动后从注册中心拉取目标服务节点列表，本地缓存节点池，实现动态服务发现。

**内网特性说明**：内网环境可信、流量封闭、访问速度极高，传统架构会弱化内网重复鉴权，追求极致性能；本架构在此基础上仍保留**服务级权限隔离**能力，兼顾性能与安全。

**负载均衡**：消费者本地基于算法（轮询/随机/加权）实现内网负载均衡，流量均摊至多实例节点。

#### 2\.2 外网前端通信：API 网关统一收口（安全可控、全网唯一入口）

针对**前后端分离外网场景**，所有前端、移动端、第三方外网调用**禁止直接访问底层微服务**，杜绝内网地址暴露、端口外泄、接口裸奔风险，全网流量统一由 API 网关唯一收口、统一管控。

- **外网标准请求格式**：`网关域名/服务名称/接口路由`

- **网关与注册中心联动**：API 网关本身也是独立微服务，同样注册到注册中心；网关实时从注册中心做服务发现，根据**服务名称路由**匹配在线健康节点，动态转发请求至底层微服务。

- **集群高可用设计**：API 网关采用**多实例集群部署**，前端前置统一负载均衡器，彻底消除网关单点故障，保障外网入口永久高可用。

#### 2\.3 API 网关统一非业务能力收口（生产必备）

网关作为整个分布式系统的**流量总入口、安全总闸门**，统一承载所有横向非业务能力，所有底层微服务无需重复开发通用逻辑，实现职责解耦。

- **统一路由转发**：根据服务名、路由规则精准分发流量至对应微服务。

- **全局认证授权**：OAuth2\.0/OIDC 令牌校验、租户校验、权限拦截、登录态校验，非法请求直接拦截，减轻后端服务压力。

- **流量管控**：接口限流、黑名单、频率控制，抵御高并发、恶意刷量流量。

- **高可用防护**：熔断、降级、故障隔离，避免服务雪崩，保障核心业务稳定。

- **可观测性**：统一日志、链路追踪、请求监控、响应耗时统计。

#### 2\.4 内外网架构分层总结

- **外网前端流量**：用户/前端 → 负载均衡 → API网关集群 → 服务发现路由 → 底层微服务（重安全、重限流、重监控）

- **内网服务流量**：微服务 ⇄ 微服务（点对点直连、去中心化、高性能、依赖注册中心发现）

该架构是互联网、支付、SaaS、企业级微服务**标准生产架构**，兼顾：**外网绝对安全 \+ 内网极致性能 \+ 全集群高可用 \+ 全服务可治理**。

重构全系统流量入口，实现**所有前端、后端微服务流量统一经过 API 网关**，打造全局统一安全防线，是企业级分布式系统通用最优解决方案。

- **全局流量收敛**：前端页面、移动端、第三方调用、微服务内部调用所有请求，统一接入 API 网关，无直接暴露后端服务端口。

- **网关前置鉴权**：将 OAuth2\.0/OIDC 令牌校验、权限拦截、租户校验、风控拦截前置至网关层，非法请求直接拦截，减少后端服务压力。

- **统一流量管控**：支持限流、熔断、黑白名单、请求日志、链路追踪，统一管控全系统接口安全与流量。

- **路由精准转发**：网关根据请求路径自动路由至 IdentityService 权限中心、业务微服务、前端静态资源服务。

### 3\. 微服务注册中心与完整服务治理体系（核心能力）

IdentityService 作为分布式系统**统一身份安全底座**，后续将深度接入主流微服务注册中心，实现全集群标准化服务治理，彻底解决传统微服务硬编码对接、服务状态不可控、调用混乱、单点故障无法自动切换的痛点，适配大规模 SaaS 微服务集群长期迭代。整套治理体系包含：**服务自动注册、动态服务发现、健康巡检、故障自动剔除、负载均衡、服务权限隔离、集群可视化治理**。

#### 3\.1 服务注册机制（Service Register）

所有微服务组件（IdentityService 权限中心、各业务微服务、后台任务服务、第三方信任服务）在启动时自动完成注册，无需人工配置，实现服务接入标准化。

- **注册核心信息**：服务名称、服务唯一标识、集群节点IP、端口、健康状态、运行环境、租户适配能力、服务权限级别、协议类型。

- **IdentityService 专属注册策略**：权限中心作为核心基础服务，注册时标记为**系统核心依赖服务**，拥有最高优先级，集群禁止批量剔除，保障全系统鉴权能力不中断。

- **多实例注册**：K3s 多 Pod 扩容场景下，多个 IdentityService 实例独立注册，形成服务集群节点列表，为负载均衡提供节点池。

- **注册校验机制**：仅通过平台管理员注册、集群白名单的合法服务可接入，防止恶意服务伪造身份接入集群、窃取权限数据。

#### 3\.2 服务发现机制（Service Discovery）

彻底摒弃传统 IP\+端口 硬编码调用方式，所有服务间通信、网关路由均基于注册中心动态发现，实现服务无感扩容、无感切换、无感更新。

- **主动拉取发现**：API 网关、各业务微服务启动后，定时拉取注册中心最新的 IdentityService 节点列表，缓存本地节点池。

- **实时推送更新**：当权限中心实例扩容、缩容、故障下线时，注册中心实时推送节点变更，下游服务自动更新调用节点，无需重启服务。

- **精准服务匹配**：支持按环境、租户、服务类型过滤节点，实现开发/测试/生产环境服务隔离，杜绝跨环境错误调用。

#### 3\.3 生产级微服务治理核心能力

- **健康检测与故障自愈**
注册中心实时心跳检测 IdentityService 及所有下游服务状态，对超时、宕机、异常报错的故障节点**自动临时剔除**，流量自动路由至健康实例；故障节点恢复后自动重新注册接入集群，实现集群自愈。

- **集群负载均衡**
针对 IdentityService 多实例部署，支持轮询、随机、加权负载均衡策略，均匀分发鉴权、令牌签发、用户信息查询请求，避免单实例压力过载，大幅提升高并发场景吞吐量。

- **服务熔断与降级**
配合网关实现权限服务熔断机制：当 IdentityService 集群压力过高、响应超时，自动触发熔断，拦截无效请求，避免服务雪崩；同时支持自定义降级策略，保障核心业务可用。

- **服务权限隔离治理**

- **服务上下线灰度治理**
支持单实例灰度上线、分批下线，配合 K3s 滚动更新，实现权限服务零停机迭代更新，不影响全系统用户登录与接口鉴权。

- **全链路监控与溯源**
记录所有服务注册、发现、调用、异常日志，统计权限服务调用量、成功率、响应耗时、异常率，实现集群治理可视化、问题快速溯源。
