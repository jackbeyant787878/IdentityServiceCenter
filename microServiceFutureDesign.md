# IdentityService 权限中心\-云原生架构展望与密钥生命周期专项技术规划

# 前言

本文档为 IdentityService 分布式权限中心**云原生架构升级、CI/CD自动化部署、RSA密钥全生命周期管理、微服务密钥平滑轮转**专项技术规划文档。基于现有DDD架构、OpenIddict鉴权体系、K3s容器部署底座，落地生产级密钥安全方案，解决私钥签发、公钥验签、密钥过期轮转、新旧密钥兼容、容器化密钥注入等核心重难点，适配整套微服务集群生产运维规范。

# 一、整体云原生架构升级规划

## 1\.1 部署底座架构

项目后续全面落地**腾讯云K3s轻量K8s集群 \+ GitHub Actions全自动CI/CD**架构，摒弃传统服务器部署模式，实现服务容器化、发布自动化、运维标准化、扩容自愈化。K3s为精简版K8s，保留容器编排、滚动更新、资源调度、Secret密钥加密、PVC持久化、服务发现核心能力，低资源消耗、易运维，完美适配本项目微服务集群。

整体部署流程：代码合并主干 → GitHub Actions自动构建Docker镜像 → 推送腾讯云TCR私有镜像仓库 → 远程K3s集群部署 → 滚动更新Pod → 零停机生产发布，支持版本追溯、一键回滚、弹性扩缩容。

## 1\.2 流量架构规范（生产最终标准）

严格遵循企业级微服务安全架构，实现**内外网流量完全隔离、外网唯一网关收口**：

- **外网流量**：前端、移动端、第三方请求仅访问API网关域名，禁止直连任何底层微服务。网关结合注册中心、K8s服务发现转发请求，路由规范：`网关域名/服务名称/接口路由`。

- **内网流量**：订单、支付、权限中心等微服务内网点对点直连，去中心化无转发损耗，兼顾高性能与内网权限隔离。

- **安全兜底**：所有底层微服务（权限中心、订单、支付）生产环境关闭外网端口暴露，仅保留K8s内网访问，杜绝裸服务安全风险。

## 1\.3 配置分层治理方案

采用**静态部署配置 \+ 动态业务配置**双层治理，解决密钥硬编码、配置散乱、生产不可热更新问题：

- **静态配置（CI/CD阶段）**：K3s密钥、镜像版本、集群权限等部署级配置，托管GitHub Secrets，无明文落地。

- **动态配置（运行时热更新）**：密钥过期时长、令牌生命周期、限流风控、租户参数，托管配置中心，支持不停机热加载、灰度推送、版本回溯。

适配\.NET10技术栈，优先选用AgileConfig轻量配置中心，无Java依赖、原生适配热更新，适配K3s容器环境。

# 二、核心重难点：RSA密钥全生命周期容器化方案

## 2\.1 公私钥分工核心设计（微服务统一鉴权标准）

基于OpenIddict原生RSA非对称加密机制，落地**私钥唯一签发、公钥全局验签**的微服务安全架构，彻底解决集群Token签发与验签一致性问题：

- **权限中心（IdentityService）**：唯一持有RSA私钥，负责全局AccessToken、RefreshToken签发、令牌加密、密钥轮转，是集群唯一鉴权签发源。

- **所有下游微服务（网关、订单、支付、业务服务）**：仅持有RSA公钥，不参与Token签发，仅用于接口请求Token验签、身份解析，最小化密钥泄露风险。

## 2\.2 K3s密钥安全存储方案（静态加密）

摒弃代码、配置文件硬编码密钥模式，采用K3s原生安全机制实现密钥静态加密存储，适配生产安全规范：

- **密钥注入方式**：将RSA签名密钥、加密密钥统一注入K3s **Secret资源**，K3s默认对Secret数据进行静态加密存储，集群内部加密留存，杜绝明文泄露。

- **持久化挂载方案**：通过**PVC持久化存储 \+ 容器挂载**模式，将密钥文件挂载至权限中心容器 `/app/keys` 目录，实现Pod重启、重建、更新后密钥不丢失。

- **权限隔离**：配置容器文件系统权限，仅授权权限中心服务读取私钥，下游微服务仅读取公钥，实现密钥分级权限管控。

## 2\.3 密钥生命周期标准化配置

结合业务安全等级与运维成本，定义生产级密钥生命周期规则，适配金融级鉴权场景：

- **签名密钥（Signing Key）**：有效期90天，用于Token签名签发，高频核心密钥，轮转周期较短。

- **加密密钥（Encryption Key）**：有效期180天，用于Token内容加密，低频密钥，延长轮转周期减少运维开销。

- **业务令牌有效期**：AccessToken 60分钟、RefreshToken 14天，兼顾用户体验与安全防泄露。

密钥过期自动触发轮转，生成全新密钥对，同时保留历史密钥，实现平滑过渡。

# 三、密钥平滑轮转核心重难点分析（源码深度解析）

## 3\.1 行业通用痛点

传统密钥轮转存在致命问题：密钥过期一刀切、新旧密钥不兼容、服务更新不同步，导致**旧Token瞬间失效、未下线旧服务验签失败、集群大面积鉴权异常**，生产事故风险极高。本项目通过源码定制改造，彻底解决该问题。

## 3\.2 核心源码实现逻辑（密钥兼容核心）

### 3\.2\.1 多密钥共存加载机制（核心重难点）

源码文件：`ConfigureOpenIddictOptions.cs` 实现了**多密钥并行注册、新旧密钥共存验签**核心能力，是平滑轮转的关键：

- 系统启动时，`LoadAllKeys()` 自动扫描PVC挂载目录，加载**当前活跃密钥 \+ 所有历史过期密钥**，不丢弃旧密钥文件。

- 服务签发Token时，**仅使用最新的活跃私钥**，保证新Token统一规范。

- 服务验签Token时，**遍历所有新旧签名密钥**，只要任意密钥验签通过即放行，完美兼容旧密钥签发的存量Token。

同时支持加密密钥多版本共存，新密钥负责加密，新旧密钥均可解密，避免轮转期间Token解密失败。

### 3\.2\.2 定时热轮转\+内存热更新机制

源码文件：`KeyRotationBackgroundService.cs`、`Program.cs` 实现全自动无感知轮转：

- 通过Hangfire定时任务，每日凌晨5点自动执行`RotateKeysAsync()`密钥轮转，生成新密钥对存入PVC持久化目录。

- 自定义`ReloadableOptions` 可重载配置，轮转完成后**原子性刷新内存OpenIddict配置**，无需重启服务、无需重建Pod，新密钥即时生效。

- 定时任务独立创建ServiceScope，规避单例生命周期异常，保证长期稳定运行。

### 3\.2\.3 手动兜底轮转能力

源码文件：`KeyManagementController.cs` 提供手动轮转接口，用于应急场景：密钥异常、紧急安全更新时，可手动触发密钥轮转，支持测试验证与生产应急兜底。

## 3\.3 容器化部署密钥轮转适配逻辑

- **新容器启动逻辑**：新Pod启动时自动加载PVC中最新密钥集合，直接继承所有新旧密钥，启动即可兼容存量Token。

- **旧服务过渡逻辑**：旧版本未下线服务，依然保留旧密钥，可正常验签旧Token；新Token由新密钥签发，旧服务通过多密钥兼容机制正常验签，无服务断层。

- **彻底消除一刀切问题**：密钥轮转只新增不删除，等待旧Token自然过期、旧服务全部滚动更新完成后，可手动清理过期历史密钥，全程零业务中断。

# 四、微服务全局密钥联动方案

## 4\.1 权限中心（签发端）

持有完整公私钥对，基于PVC挂载动态加载密钥，定时自动轮转、内存热更新，唯一负责全局Token签发、加密、密钥迭代。

## 4\.2 下游微服务（验签端）

网关、订单、支付等所有业务服务，仅注入公钥文件，启动加载公钥集合。依托OpenIddict Validation机制，自动兼容新旧公钥，无需业务改造，无感适配密钥轮转。

# 五、生产落地注意事项与避坑规范

- **禁止直接删除历史密钥**：历史密钥是兼容旧Token、旧服务的核心，必须等待Token过期、服务全量更新后清理。

- **密钥持久化不可中断**：PVC存储为密钥安全核心，禁止随意删除、卸载PV资源，防止密钥丢失导致全局验签瘫痪。

- **严格区分公私钥权限**：私钥仅权限中心持有，禁止下发下游微服务，最小化安全风险。

- **轮转时间错开业务高峰**：固定凌晨低峰期轮转，避免高并发场景下密钥更新引发短暂抖动。

- **支持灰度迭代**：配合K3s滚动更新策略，服务分批更新，新旧服务共存过渡，保障集群稳定。

# 六、架构迭代价值总结

- **安全可控**：K3s Secret静态加密\+PVC持久化\+公私钥分离，实现金融级密钥安全管控。

- **高可用无中断**：多密钥共存、平滑轮转、热更新机制，彻底解决密钥迭代业务中断问题。

- **运维自动化**：全自动定时轮转、CI/CD自动部署，减少人工运维成本与失误风险。

- **架构标准化**：完全适配云原生微服务规范，可无缝拓展更多业务微服务，扩展性极强。
